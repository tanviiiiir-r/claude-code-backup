## Learned Patterns

### Common Red Flags Detected

#### Experience Mismatches
- "Entry-level" role requiring 3+ years experience
- "Junior" position with senior-level responsibilities
- Master's degree required for junior pay
- PhD preferred for implementation roles

#### Vague Requirements
- "Other duties as assigned" appearing 3+ times
- No specific tech stack mentioned
- "Willing to wear multiple hats" (scope creep indicator)
- "Fast-paced environment" without specifics

#### Cultural Red Flags
- "We're a family" (often poor boundaries)
- "Work hard, play hard" (unsustainable pace)
- "Ninja/Rockstar/Guru" titles (unclear expectations)
- "Unlimited PTO" (often means no PTO taken)
- "High-energy environment" (burnout risk)

### Green Flags Learned

#### Positive Indicators
- Mentorship programs explicitly mentioned
- Professional development budget specified
- Clear growth paths documented
- Work-life balance as core value
- Remote-first or hybrid policies
- Technical blog presence
- Open source contributions

### Skills Analysis Patterns

#### Must-Have vs Nice-to-Have Detection
- **Must-have:** Listed in "Requirements", "Required", "Essential"
- **Nice-to-have:** Listed in "Preferred", "Bonus", "Ideal", "Plus"
- **Ambiguous:** Listed without qualifier (treat as must-have)

#### Scope Indicators
| Indicator | Meaning |
|-----------|---------|
| "Full stack" + specific backend | Frontend secondary |
| "Full stack" + specific frontend | Backend secondary |
| "DevOps experience a plus" | You'll own infrastructure |
| "Customer-facing" | Support duties included |

## Company-Specific Patterns

### Nordic Companies (Finland/Sweden)
- Typically mention flat hierarchy
- Emphasize work-life balance
- Often have 6 weeks PTO
- May require Swedish/Finnish language

### Startups (500-1000 employees)
- Fast hiring process
- Equity mentioned early
- "Impact" and "ownership" emphasized
- May lack structured onboarding

### Corporates (Ericsson-style)
- Multi-round interview process
- Structured career ladders
- Benefits standardized
- Slower decision-making

## JD Structure Analysis

### Well-Written JDs Include
1. Clear role level (Junior/Mid/Senior)
2. Specific tech stack with versions
3. Concrete responsibilities (not vague)
4. Salary range (EU requirement)
5. Interview process outline
6. Team structure description

### Poor JD Indicators
1. Copy-paste from 5 years ago
2. Internal jargon without explanation
3. "Rockstar/Ninja/Guru" language
4. Excessive requirements list
5. No mention of benefits

## Application Strategy Learned

### CV Tailoring Priorities
1. Match exact keywords from JD (ATS optimization)
2. Quantify achievements matching responsibilities
3. Address gaps proactively in cover letter
4. Highlight relevant projects first

### Cover Letter Angles That Work
- **Startup:** Emphasize speed, adaptability, ownership
- **Corporate:** Emphasize collaboration, process, scale
- **AI/ML:** Emphasize publications, Kaggle, open source
- **Remote:** Emphasize async communication, documentation

### Questions That Reveal Culture
1. "What's the onboarding process like?"
2. "How do teams handle technical debt?"
3. "What's the typical career progression?"
4. "How do you measure success in this role?"
